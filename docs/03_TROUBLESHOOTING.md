# 문제 해결 가이드

> 문제가 생겼을 때 이 문서를 보고 따라하세요.
> 위에서부터 순서대로 시도하면 대부분 해결됩니다.

---

## 가장 중요한 원칙

1. **실험 중이면 2분 기다려 보세요.** 대부분 자동 복구됩니다.
2. **영상이 돌고 있으면 절대 건드리지 마세요.** 워치만 끊긴 거면 Flash에 백업 중입니다.
3. **모르겠으면 준영이한테 연락하세요.** 원격으로 도와줄 수 있습니다.

## 문제 1: "센싱 시작이 안 돼요"

### 증상
- start.sh 실행했는데 에러가 나옴
- "동글 안 잡힘" 메시지

### 해결 (순서대로 시도)

**1단계: USB 케이블 확인**
- USB 허브에 전원이 들어와 있나요? (LED 확인)
- 케이블이 빠져있으면 꽂아주세요
- 꽂은 후 **10초 기다리세요**

**2단계: 워치 확인**
- 워치 화면을 **터치해서 깨워주세요**
- 충전 크래들에 있으면 **빼주세요**

**3단계: 다시 시작**
```bash
./ops/start.sh C040
```

**4단계: 그래도 안 되면 — 컴퓨터 재시작**
```bash
sudo reboot
```
재시작 후 2분 기다린 다음, USB를 꽂고 다시 시작하세요.

## 문제 2: "영상이 안 나와요" (video_main.mp4 없음)

### 원인
- 카메라 USB가 빠졌거나 인식 안 됨

### 해결

**1단계: 카메라 USB 확인**
```bash
ls /dev/video*
```
- `/dev/video0` 같은 게 나오면 카메라는 인식됨
- 아무것도 안 나오면 → 2단계

**2단계: USB 리셋**
```bash
# Thor 기준
sudo sh -c 'echo a80aa10000.usb > /sys/bus/platform/drivers/tegra-xusb/unbind'
sleep 3
sudo sh -c 'echo a80aa10000.usb > /sys/bus/platform/drivers/tegra-xusb/bind'
sleep 5
```

**3단계: 센싱 재시작**
```bash
./ops/stop.sh
./ops/start.sh C040
```

## 문제 3: "워치 데이터가 안 나와요" (ppg.csv 없음)

### 원인
- 워치가 잠자기 모드 → BLE 연결 안 됨
- 동글이 잠김

### 해결

**1단계: 2분 기다리기**
- monitor_ble2.sh가 자동으로 재연결합니다
- 2분 안에 복구되는 경우가 많습니다

**2단계: 워치 깨우기**
- 워치 화면을 **손가락으로 터치**
- 화면이 켜지는지 확인

**3단계: 그래도 안 되면 — 워치 리셋**
- 워치를 **충전 크래들에 5초 놨다가 빼기** (하드 리셋)
- 다시 손목에 채우기
- 30초 기다리기

**4단계: 센싱 재시작**
```bash
./ops/stop.sh
./ops/start.sh C040
```

> **중요:** 워치 데이터가 BLE로 안 와도, 워치 내부 Flash에 자동 기록됩니다.
> 실험 후에 `python3 recovery/flash_download.py`로 복구할 수 있습니다.
> **실험을 중단할 필요는 없습니다!**

## 문제 4: "GSR(피부전도)만 안 나와요"

### 이건 알려진 이슈입니다

- EDA(GSR) BLE 스트림이 간헐적으로 안 나오는 현상
- **워치 Flash에는 기록 중**이므로 데이터 자체는 보존됩니다
- 실험을 중단하지 마세요

### 실험 후 복구 방법
```bash
# 워치를 충전 크래들에 꽂고:
python3 recovery/flash_download.py
```

## 문제 5: "디스크가 꽉 찼어요"

### 증상
- start.sh에서 "디스크 10GB 미만" 에러
- 또는 실험 중 갑자기 녹화가 멈춤

### 해결

**1단계: 현재 공간 확인**
```bash
df -h /
```

**2단계: 이전 참가자 데이터 백업 + 삭제**
```bash
# 외장하드로 복사 (외장하드가 연결되어 있다면)
cp -r data/C035 /media/jetson/외장하드/

# 복사 확인
ls /media/jetson/외장하드/C035/ | wc -l    # 폴더 수 확인
ls data/C035/ | wc -l                       # 원본 폴더 수 확인
# 두 숫자가 같으면 OK

# 확인 후 삭제
rm -rf data/C035
```

> ⚠️ **삭제 전 반드시 복사가 완료되었는지 확인하세요!**
> 폴더 수가 같은지 비교하세요.

## 문제 6: "컴퓨터가 먹통이에요" (SSH 안 됨)

### 원인
- Jetson 네트워크 문제 또는 시스템 행

### 해결 (현장에서만 가능)

**1단계: 모니터 확인**
- 화면이 나오면 → 네트워크만 문제. 5분 기다려보기

**2단계: 강제 재시작**
1. USB 허브(카메라/동글/마이크) **전부 뽑기**
2. 전원 버튼을 **10초 이상 누르기** (강제 종료)
3. 10초 기다리기
4. 전원 버튼 다시 눌러서 켜기
5. 바탕화면이 나올 때까지 기다리기 (1~2분)
6. USB 허브 꽂기
7. 10초 기다리기
8. 센싱 시작

## 문제 7: "이상한 에러 메시지가 나와요"

### 흔한 메시지들 (무시해도 됩니다)

| 메시지 | 의미 | 대응 |
|--------|------|------|
| `LIBUSB_ERROR_BUSY` | 동글이 일시적으로 잠김 | 자동 복구됨, 기다리기 |
| `Device or resource busy` | 카메라 일시 점유 | 자동 복구됨, 기다리기 |
| `[Audio] status: input overflow` | 오디오 버퍼 일시 넘침 | 정상 범위, 무시 |
| `GStreamer warning` | 영상 인코더 경고 | 대부분 무시 가능 |
| `BLE connection timeout` | 워치 연결 실패 | monitor_ble2가 재시도 |

### 심각한 메시지 (연락 필요)

| 메시지 | 의미 | 대응 |
|--------|------|------|
| `Segmentation fault` | 프로그램 크래시 | 센싱 재시작 필요 |
| `No space left on device` | 디스크 꽉 참 | 즉시 데이터 정리 |
| `Permission denied` | 권한 문제 | 준영이에게 연락 |
