# 데이터 검증 가이드

> 실험이 끝나면 **현장을 떠나기 전에** 반드시 검증하세요.
> 나중에 데이터가 깨진 걸 발견하면 다시 실험할 수 없습니다.

---

## 왜 검증이 필요한가?

| 문제 | 발생 빈도 | 발견 시점 | 결과 |
|------|----------|----------|------|
| 영상이 안 끊겨서 MP4 손상 | 가끔 | 분석할 때 | 해당 구간 영상 없음 |
| 워치가 중간에 끊겼다 연결됨 | 자주 | 데이터 확인 시 | PPG 갭 발생 |
| 마이크 전원이 꺼져있었음 | 가끔 | 분석할 때 | 해당 구간 음성 없음 |
| 디스크 풀로 저장 중단 | 드물게 | 다음 실험 시작 시 | 이후 데이터 전부 없음 |

**검증 스크립트를 돌리면 이 모든 문제를 1분 안에 잡아냅니다.**

## 검증 방법 (복사 붙여넣기)

### 기본 검증
```bash
cd ~/Desktop/sensing-collector
python3 validate/validate_session.py data/C040
```

### 결과 파일로 저장 (나중에 보고할 때)
```bash
python3 validate/validate_session.py data/C040 --report reports/C040_validation.json
```

## 검증 항목 설명

| 검증 | 뭘 확인하나 | 통과 기준 |
|------|-----------|----------|
| 파일 존재 | 매 1분 폴더에 모든 센서 파일이 있는지 | 5개 모달리티 전부 존재 |
| 영상 크기 | video_main/sub.mp4이 충분한 크기인지 | 5MB 이상 |
| 영상 무결성 | MP4에 moov atom이 있는지 (정상 종료됐는지) | moov 존재 |
| 음성 무결성 | WAV 파일 헤더가 정상인지 | RIFF+WAVE 헤더 |
| CSV 파싱 | ppg/gsr/temp CSV가 읽을 수 있는지 | 파싱 가능 + 5행 이상 |
| 타임스탬프 | PPG 데이터에 큰 갭이 없는지 | 2분 이상 갭 없음 |

## 교차 검증 (Flash vs BLE)

워치 데이터가 BLE로 제대로 왔는지 Flash 데이터와 비교할 수 있습니다.

### 1단계: Flash 데이터 다운로드
```bash
# 워치를 충전 크래들에 꽂고:
python3 recovery/flash_download.py
```

### 2단계: 비교
```bash
# BLE CSV의 PPG 행 수
wc -l data/C040/*/ppg.csv | tail -1

# Flash CSV의 PPG 행 수
wc -l flash_data/ppg_flash.csv
```

두 숫자가 **비슷하면** (±5% 이내) BLE 전송이 정상이었습니다.
Flash가 훨씬 많으면, BLE가 끊긴 구간이 있었다는 뜻입니다.
→ Flash 데이터를 사용하면 됩니다.
