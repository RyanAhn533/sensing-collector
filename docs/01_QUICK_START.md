# 센싱 시스템 빠른 시작 가이드

> 이 문서는 컴퓨터에 익숙하지 않은 분도 따라할 수 있도록 작성되었습니다.
> 모든 명령어는 **그대로 복사해서 붙여넣기**하면 됩니다.

---

## 전체 흐름 (5분이면 됩니다)

```
준비 (2분) → 시작 (1분) → 확인 (1분) → 실험 진행 → 종료 (1분)
```

---

## 1단계: 장비 준비 (모니터 앞에서)

### 워치 준비
1. **충전 크래들에서 워치를 빼세요** (크래들에 있으면 연결 안 됩니다)
2. **워치 화면을 손가락으로 터치**해서 화면이 켜지는지 확인하세요
3. 워치를 **피실험자 손목에 채워주세요** (밴드가 피부에 밀착되어야 합니다)

### 마이크 준비
1. RODE TX(작은 송신기)의 **전원 버튼을 눌러서 켜세요** (LED 빛남)
2. 피실험자의 **옷 앞쪽에 클립으로 고정**하세요

### 카메라 확인
1. 정면 카메라가 **운전자 얼굴을 향하고 있는지** 확인
2. 측면 카메라가 **운전자 상반신을 향하고 있는지** 확인

---

## 2단계: 센싱 시작

### 방법 A: 바탕화면 버튼 (가장 쉬움)
1. Jetson 모니터에서 **[센싱 시작]** 아이콘을 더블클릭
2. 참가자 번호를 입력하라고 나오면 숫자 입력 (예: `C040`)
3. **엔터** 누르기

### 방법 B: 터미널 (원격으로 할 때)
```bash
# 1. 터미널을 열거나 SSH로 접속합니다
ssh jetson@165.132.48.241
# 비밀번호: jetson

# 2. 센싱 폴더로 이동합니다
cd ~/Desktop/sensing-collector

# 3. 센싱을 시작합니다 (C040은 참가자 번호입니다. 바꿔주세요)
./ops/start.sh C040
```

### 시작 후 화면에 이런 메시지가 나옵니다:
```
[10:15:03] ========== PRECHECK ==========
[10:15:03] 동글: /dev/ttyACM0
[10:15:03] 디스크 여유: 54GB
[10:15:03] 참가자: C040
[10:15:03] ========== SENSING START ==========
[10:15:08] main.py 시작 (PID: 12345)
[10:15:43] ========== VALIDATION (35s wait) ==========
[10:15:43] 센서 상태: v=OK a=OK ppg=OK gsr=OK temp=OK   ← 전부 OK면 성공!
[10:15:43] ========== READY ==========
```

### 센서 상태 읽는 법

| 표시 | 의미 | OK가 아닐 때 |
|------|------|-------------|
| `v=OK` | 영상 녹화 중 | 카메라 USB 확인 |
| `a=OK` | 음성 녹음 중 | 마이크 전원 확인 |
| `ppg=OK` | 심박 측정 중 | 워치 화면 터치 후 재시작 |
| `gsr=OK` | 피부전도 측정 중 | 안 나와도 Flash에 기록됨 (걱정 마세요) |
| `temp=OK` | 피부온도 측정 중 | 워치 밀착 확인 |

**v=OK이고 ppg=OK이면 실험 진행해도 됩니다.**
gsr이 X여도 워치 Flash에 자동 백업되니까 데이터는 살아있습니다.

---

## 3단계: 실험 진행

### 실험 중에 해야 할 것
- **아무것도 안 해도 됩니다!** 자동으로 모니터링됩니다.
- 가끔 터미널에 로그가 나오는 건 정상입니다.

### 실험 중에 하면 안 되는 것
- ❌ USB 케이블 건드리기
- ❌ 키보드에서 Ctrl+C 누르기
- ❌ 터미널 창 닫기
- ❌ 컴퓨터 끄기/재시작

### 문제가 생긴 것 같을 때
대부분의 문제는 **자동으로 복구**됩니다. 2분 기다려 보세요.
그래도 안 되면 → "문제 해결 가이드" (`03_TROUBLESHOOTING.md`) 참고

---

## 4단계: 센싱 종료

### 방법 A: 바탕화면 버튼
1. **[센싱 종료]** 아이콘을 더블클릭

### 방법 B: 터미널
```bash
cd ~/Desktop/sensing-collector
./ops/stop.sh
```

---

## 5단계: 데이터 확인 (중요!)

**실험이 끝나면 반드시 확인하세요. 1분이면 됩니다.**

```bash
python3 validate/validate_session.py data/C040
```

### 결과 해석:

**PASS** = 데이터 정상. 다음 실험 가능.
```
VERDICT: PASS
```

**PASS_WITH_WARNINGS** = 대부분 정상. 일부 누락 있지만 사용 가능.
```
VERDICT: PASS_WITH_WARNINGS
Missing modalities:
  gsr: 3 minutes missing    ← GSR 3분 빠짐 (Flash에는 있음)
```

**FAIL** = 문제 있음. 로그 확인 필요.
```
VERDICT: FAIL
Issues:
  - 20260327_1015: video_main: MP4 moov atom missing   ← 영상 손상
```
